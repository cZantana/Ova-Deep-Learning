Overfitting, Underfitting y Regularización en Deep Learning

Overfitting (sobreajuste), Underfitting (infraajuste) y Regularización son conceptos fundamentales en el aprendizaje automático y el deep learning. Son cruciales para comprender cómo los modelos de deep learning aprenden de los datos y cómo generalizan a nuevos datos no vistos. A continuación, se

proporciona una definición completa de cada uno, la matemática subyacente, su propósito, la necesidad de cada uno y su utilidad en diferentes contextos.

¿Qué es el Underfitting?

Underfitting se refiere a una situación en la que un modelo de machine learning no puede aprender adecuadamente a partir de los datos de entrenamiento. Esto sucede cuando el modelo es demasiado simple, lo que significa que tiene una capacidad limitada para capturar la complejidad de los datos. En otras palabras, el modelo "subestima" la relación entre las características de entrada y la variable de salida, resultando en predicciones imprecisas.


 

Imagen tomada de: Overfitting and underfitting in machine learning | SuperAnnotate

Características del Underfitting

  1. Modelo Demasiado Simple: Un modelo que sufre de underfitting generalmente tiene una estructura o capacidad limitada. Esto podría deberse a un número insuficiente de parámetros (por ejemplo, pocas neuronas en una red neuronal o un polinomio de bajo grado en la regresión polinómica) o a la elección de un modelo que no es adecuado para los datos (por ejemplo, usar un modelo lineal para datos no lineales).

  2. Bias Alto: Un modelo que subestima la relación en los datos generalmente tiene un bias (sesgo) alto. El bias alto indica que el modelo es incapaz de captar la complejidad de los datos y hace suposiciones demasiado simplificadas sobre la relación entre las entradas y la salida.

  3. Rendimiento Deficiente en Datos de Entrenamiento y Prueba: A diferencia del overfitting, donde el modelo tiene un error de entrenamiento bajo, pero un error de prueba alto en el underfitting, el modelo tiene un error alto tanto en los datos de entrenamiento como en los de prueba. Esto se debe a que el modelo no ha aprendido correctamente la estructura de los datos.

  4. Generalización Insuficiente: Un modelo que sufre de underfitting no generaliza bien porque no aprende lo suficiente de los datos de entrenamiento. Esto significa que el modelo tiene un rendimiento consistentemente pobre en cualquier conjunto de datos, ya que no ha captado los patrones subyacentes.

¿Por Qué se Produce el Underfitting?

El underfitting ocurre por varias razones:

  1. Modelo Insuficientemente Complejo: Si la estructura del modelo es demasiado simple para los datos, como usar una línea recta para un conjunto de datos que sigue una curva compleja, el modelo no tendrá la capacidad de ajustarse adecuadamente.
  2. Demasiada Regularización: La regularización es una técnica para evitar el overfitting, pero si se aplica en exceso, puede hacer que el modelo se vuelva demasiado simple y conduzca al underfitting.
  3. Pocos Datos de Entrenamiento: Si el modelo no tiene suficientes ejemplos de los cuales aprender, puede ser incapaz de capturar la variabilidad y los patrones complejos en los datos.
  4. Hiperparámetros Mal Ajustados: La elección incorrecta de hiperparámetros, como una tasa de aprendizaje demasiado alta que impide la convergencia adecuada, puede llevar al underfitting.

 

 

Cómo Evitar el Underfitting

  1. Aumentar la Complejidad del Modelo: Usar un modelo más complejo que tenga la capacidad de aprender patrones más intrincados en los datos. Por ejemplo, aumentar el número de neuronas o capas en una red neuronal.

  2. Reducir la Regularización: Si se está utilizando regularización, puede ser útil reducir su intensidad para permitir que el modelo se ajuste más a los datos.

  3. Aumentar el Número de Épocas de Entrenamiento: Entrenar el modelo durante más tiempo para asegurarse de que tenga la oportunidad de aprender suficientemente de los datos.

  4. Añadir Más Características Relevantes: Incluir más características que sean relevantes para la tarea de aprendizaje puede ayudar al modelo a capturar mejor la relación entre las variables de entrada y salida.

¿Qué es Overfitting?

Overfitting, o sobreajuste, ocurre cuando un modelo de aprendizaje automático se ajusta demasiado a los datos de entrenamiento, capturando tanto los patrones reales como el ruido o las peculiaridades de esos datos específicos. Como resultado, el modelo aprende a memorizar los datos de entrenamiento en lugar de generalizar patrones a partir de ellos. Esto lleva a un rendimiento pobre cuando el modelo se enfrenta a nuevos datos que no ha visto antes.

Matemáticamente, el sobre ajuste puede entenderse mediante el concepto de error de generalización, que es la diferencia entre el error en los datos de entrenamiento y el error en los datos de prueba. El error total de un modelo puede dividirse en tres componentes:

  1. Error Irreducible: Error inherente a cualquier modelo debido a la variabilidad natural en los datos.

  2. Bias (Sesgo): Error debido a suposiciones simplificadas en el modelo. Un modelo con high bias (Alto sesgo) subestima la relación entre las variables de entrada y salida, resultando en un error de entrenamiento alto.

  3. Variance (Varianza): Error debido a la sensibilidad del modelo a las fluctuaciones en el conjunto de datos de entrenamiento. Un modelo con high variance (Varianza alta) puede tener un error de entrenamiento bajo pero un error de prueba alto porque se ajusta demasiado a los datos de entrenamiento.

El sobreajuste ocurre cuando el bias es bajo y la varianza es alta, es decir, cuando el modelo se ajusta muy bien a los datos de entrenamiento, pero no generaliza bien a los datos nuevos.

¿Por Qué se Produce el Overfitting?

  1. Modelo Demasiado Complejo para los Datos:

Cuando la complejidad del modelo supera la cantidad o la variabilidad de los datos de entrenamiento, el modelo tiene suficiente flexibilidad para aprender detalles específicos de los datos de entrenamiento, incluyendo ruido y outliers.

  1. Cantidad Insuficiente de Datos de Entrenamiento:

Con un número limitado de muestras de entrenamiento, es más probable que el modelo aprenda detalles específicos de esas muestras en lugar de patrones generalizables.

  1. Entrenamiento Demasiado Prolongado:

Si el modelo se entrena durante demasiadas épocas, especialmente en redes neuronales, puede comenzar a aprender patrones muy específicos a los datos de entrenamiento en lugar de características generales.

  1. Falta de Regularización:

La ausencia de técnicas de regularización permite que los modelos sean tan complejos como lo deseen para minimizar el error de entrenamiento, lo que aumenta el riesgo de overfitting.

Cómo Evitar el Overfitting

  1. Aumentar el Tamaño del Conjunto de Datos de Entrenamiento:

○     Más datos permiten al modelo aprender patrones generales en lugar de detalles específicos de ejemplos individuales. Las técnicas de data augmentation pueden ser útiles en problemas como la clasificación de imágenes, donde se pueden generar nuevas muestras a partir de las existentes mediante transformaciones como rotación, cambio de escala, etc.

  1. Regularización:

○     L2 Regularization (Ridge): Penaliza los pesos grandes, empujándolos a valores más pequeños y promoviendo un modelo más simple.

○     L1 Regularization (Lasso): Promueve la escasez de características al penalizar la suma de los valores absolutos de los pesos, empujando muchos de ellos a cero.

○     Dropout: Durante el entrenamiento, apaga aleatoriamente una fracción de las neuronas en cada capa. Esto evita que las neuronas dependan demasiado unas de otras y fuerza a la red a aprender representaciones más robustas.

  1. Early Stopping:

○     Monitorea el error de validación durante el entrenamiento y detiene el proceso cuando el error de validación deja de mejorar, evitando que el modelo se ajuste demasiado a los datos de entrenamiento.

  1. Simplificar el Modelo:

○     Reducir la complejidad del modelo puede ayudar a evitar el overfitting. Esto puede implicar reducir el número de capas o neuronas en una red neuronal, reducir el grado de un polinomio en la regresión polinómica, o podar árboles de decisión para limitar su profundidad.

  1. Cross-Validation (Validación Cruzada):

○     Dividir los datos en múltiples pliegues y entrenar varios modelos para cada subconjunto de datos. Esto ayuda a asegurar que el modelo generalice bien en diferentes partes del conjunto de datos y no dependa demasiado de un solo subconjunto.

  1. Regularización de Peso Decaído (Weight Decay):

○     Una técnica específica para optimizadores como Adam, que incorpora un término de decaimiento del peso en la actualización de los pesos, similar a L2 regularization, para penalizar pesos grandes.

  1. Ensemble Methods (Métodos de Ensamble):

○     Técnicas como bagging (Bootstrap Aggregating) y boosting combinan múltiples modelos para reducir la varianza y mejorar la generalización del modelo. Random Forest es un ejemplo de bagging, mientras que AdaBoost es un ejemplo de boosting.

 

Ahora bien, ya que se entienden estos dos conceptos, vamos a ver las técnicas de regularización, para reducir al máximo, o, evitar por completo, el underfitting o el overfitting, y su efecto en el entrenamiento de redes neuronales.

¿Qué es la Regularización?

Regularización es un conjunto de técnicas utilizadas para prevenir el sobreajuste penalizando la complejidad del modelo. La idea básica es añadir un término de regularización a la función de pérdida que desaliente a la red de ajustarse demasiado a los datos de entrenamiento. Este término penaliza los parámetros del modelo (como los pesos de una red neuronal) que son grandes en magnitud, incentivando soluciones más simples y evitando que el modelo se ajuste demasiado a ruidos específicos de los datos de entrenamiento. La regularización se implementa, generalmente, añadiendo un término de penalización a la función de pérdida. Los métodos más comunes de regularización en deep learning son: L2 regularization, L1 regularization, Dropout, Data Augmentation y Early Stopping

 L2 Regularization (Ridge Regularization)

Contexto y Definición

L2 Regularization, también conocida como Ridge Regularization, es una técnica de regularización que penaliza los pesos grandes al añadir un término proporcional al cuadrado de los pesos a la función de pérdida del modelo. La idea principal es evitar que los parámetros del modelo tomen valores extremadamente altos, lo que puede llevar a un sobreajuste.

L1 Regularization (Lasso Regularization)

Contexto y Definición

L1 Regularization, también conocida como Lasso Regularization (Least Absolute Shrinkage and Selection Operator), es una técnica de regularización que penaliza los pesos grandes al añadir un término proporcional al valor absoluto de los pesos a la función de pérdida del modelo. A diferencia de L2, L1 regularization tiende a hacer que algunos pesos sean exactamente cero, lo que puede resultar en un modelo más simple y escaso.

Al forzar algunos pesos a ser exactamente cero, L1 regularización efectivamente elimina algunas características del modelo, haciendo que sea más fácil de interpretar y más eficiente computacionalmente.

Contexto de Utilidad

●     Selección de Características: Útil en problemas de alta dimensionalidad para seleccionar un subconjunto de características relevantes.

●     Modelos Escasos: Adecuado cuando se espera que solo unas pocas características sean importantes.

 

Dropout

Contexto y Definición

Dropout es una técnica de regularización específicamente diseñada para redes neuronales. Durante el entrenamiento, dropout apaga aleatoriamente un porcentaje de neuronas en cada capa de la red en cada iteración, lo que impide que las neuronas se vuelvan demasiado dependientes unas de otras.

Contexto de Utilidad

Data Augmentation

Contexto y Definición

Data Augmentation es una técnica de regularización que aumenta artificialmente el tamaño del conjunto de datos de entrenamiento aplicando transformaciones a los datos existentes. Esto incluye operaciones como rotación, traslación, escalado, volteo y adición de ruido, entre otras.

Contexto de Utilidad

●     Visión por Computadora: Es ampliamente utilizado en tareas de clasificación de imágenes y detección de objetos, donde las transformaciones visuales pueden generar muchas variaciones del mismo dato.

●     Reconocimiento de Voz: Data augmentation también se utiliza en el procesamiento de señales de audio, como en el reconocimiento de voz.

Early Stopping

Contexto y Definición

Early Stopping es una técnica de regularización que detiene el entrenamiento de un modelo cuando el rendimiento en el conjunto de validación deja de mejorar. La idea es detener el entrenamiento antes de que el modelo comience a sobreajustarse a los datos de entrenamiento.

 

 

Matemática detrás de Early Stopping

Early stopping no implica cambios en la función de pérdida, pero su implementación práctica se basa en la monitorización del error de validación ():

  1. Durante el entrenamiento, después de cada época, se evalúa el modelo en un conjunto de validación.
  2. Si no mejora después de un número definido de épocas consecutivas, el entrenamiento se detiene.

¿Para Qué Sirve?

Early stopping ayuda a prevenir el overfitting al detener el entrenamiento en el punto en el que el modelo comienza a sobreajustarse a los datos de entrenamiento, maximizando así su rendimiento en datos no vistos.

Contexto de Utilidad

●     Entrenamiento de Redes Neuronales: Muy útil cuando el entrenamiento es largo y costoso, ya que evita gastar tiempo y recursos entrenando un modelo más allá del punto óptimo.

●     Modelos de Alto Riesgo: En aplicaciones donde es crucial evitar el sobreajuste debido a la importancia de la precisión del modelo, como en medicina o finanzas.

Cada una de estas técnicas de regularización: L2 regularization, L1 regularization, Dropout, Data Augmentation y Early Stopping, juega un papel crucial en la mejora de la generalización de los modelos de deep learning y en la reducción del riesgo de overfitting. Al entender cómo y cuándo aplicar cada técnica, los practicantes de deep learning pueden desarrollar modelos más robustos y efectivos que generalicen bien a datos no vistos, lo que es esencial para el éxito en aplicaciones del mundo real.
