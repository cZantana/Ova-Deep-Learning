Que son los Notebooks y que es Google Collab?

 

Antes de iniciar la programación de un perceptrón, una red neuronal de una sola capa o una red neuronal multicapa, es importante familiarizarse con las herramientas que se utilizaran. Una de las herramientas clave en este curso serán los notebooks de Python.

 

Notebooks de Python

 

Un notebook de Python es un entorno interactivo de desarrollo que permite escribir y ejecutar código en pequeños bloques, llamados celdas. Cada celda puede contener código, texto en formato Markdown, ecuaciones matemáticas, gráficos y visualizaciones. Este enfoque modular facilita la experimentación y el aprendizaje, permitiendo ejecutar y probar partes del código de manera incremental.

 

 

Imagen tomada de: 28 Jupyter Notebook Tips, Tricks, and Shortcuts - Dataquest

 

Diferencias de un Notebook con un Script de Python común:

●     Interactividad: Los notebooks permiten ejecutar celdas individuales de código, facilitando la prueba y el ajuste de pequeñas secciones del código, mientras que un script común, se ejecuta de principio a fin de una sola vez.

●     Documentación y Visualización: En los notebooks, se puede intercalar código con texto explicativo y visualizaciones, proporcionando una documentación clara y visual directamente en el flujo de trabajo. En un script de Python, la documentación suele ser menos interactiva y más lineal.

●     Facilidad de Uso para la Enseñanza: Los notebooks son especialmente útiles en entornos educativos, ya que permiten a los estudiantes experimentar con el código y ver resultados inmediatos, facilitando la comprensión de conceptos complejos.

Interfaces Populares

●     Jupyter Notebook: Es una de las interfaces más populares para trabajar con notebooks de Python. Permite crear y compartir documentos que contienen código en vivo, ecuaciones, visualizaciones y texto explicativo.

●     Google Colab: Es una herramienta gratuita de Google que permite escribir y ejecutar código Python en el navegador. Ofrece varias ventajas:

-       Accesibilidad: Al estar basada en la nube, no es necesario instalar nada en el ordenador local.

-       Recursos de Computación: Ofrece acceso a GPU y TPU, lo que es beneficioso para entrenar modelos de Deep Learning.

-       Facilidad de Colaboración: Los notebooks pueden ser compartidos y editados colaborativamente, similar a cómo se trabaja con documentos de Google Docs.

Para esta ocasión, todas las actividades de programación siguientes se realizarán utilizando Google Colab. Esta plataforma nos permitirá escribir y ejecutar código Python en un entorno interactivo y colaborativo, aprovechando los recursos de computación en la nube, como GPUs, para facilitar y acelerar el desarrollo y la experimentación con perceptrones y redes neuronales.

Que es Git y GitHub?

Para este curso, además de adentrarse y familiarizarse con Google Colab, los estudiantes deberán tener conocimientos de Git y GitHub, ya que estas herramientas serán cruciales para gestionar el código y colaborar eficientemente en las actividades de programación.

Imagen tomada de: Git y Github ¿Cuáles son sus diferencias? * Consultores Expertos

Git es un sistema de control de versiones distribuido, diseñado para manejar todo tipo de proyectos, desde pequeños hasta muy grandes, con rapidez y eficiencia. Fue creado por Linus Torvalds en 2005 para el desarrollo del kernel de Linux. Git permite a los desarrolladores:

●     Rastrear Cambios: Mantener un registro detallado de los cambios realizados en el código a lo largo del tiempo.

●     Colaborar: Trabajar en paralelo con otros desarrolladores en el mismo proyecto sin interferir con el trabajo de los demás.

●     Ramas y Fusión: Crear ramas (branch) para desarrollar nuevas características o corregir errores de manera aislada, y luego fusionarlas (merge) con la rama principal una vez que estén completas y probadas.

GitHub es una plataforma basada en la web que utiliza Git para el control de versiones y la colaboración en proyectos de software. GitHub proporciona un entorno amigable y muchas herramientas adicionales para facilitar el desarrollo colaborativo. Entre sus características destacan:

●     Repositorios: Almacenan el código fuente y su historial de versiones.

●     Colaboración: Permiten a los equipos de desarrollo colaborar en proyectos, realizar revisiones de código y gestionar solicitudes de cambios (pull requests).

●     Integración Continua: Soporta integración con otras herramientas de desarrollo y despliegue continuo.

●     Documentación y Wikis: Facilita la creación de documentación para proyectos.

Comandos mas utilizados de git:

 

Comando

Descripción

git init

Inicializa un nuevo repositorio Git en el directorio actual.

git clone <url>

Clona un repositorio remoto en tu máquina local.

git status

Muestra el estado de los archivos en el área de trabajo y el área de preparación (staging area).

git add <archivo>       

Añade un archivo o directorio al área de preparación para ser confirmado (staged).

git commit -m "mensaje"        

Crea un commit con los cambios en el área de preparación y agrega un mensaje descriptivo.

git push          

Sube los commits locales al repositorio remoto.

git pull 

Actualiza el repositorio local con los últimos cambios del repositorio remoto.

git fetch          

Descarga los últimos cambios del repositorio remoto sin fusionarlos automáticamente.

git merge <rama>      

Fusiona la rama especificada con la rama actual.

git checkout <rama> 

Cambia a la rama especificada o restaura archivos de una rama o commit.

git branch       

Muestra una lista de todas las ramas en el repositorio.

git branch <nombre> 

Crea una nueva rama con el nombre especificado.

git log 

Muestra el historial de commits del repositorio.

git remote       

Muestra los repositorios remotos configurados.

git remote add <nombre> <url>        

Agrega un nuevo repositorio remoto con un nombre asociado.

git reset <archivo>     

Quita un archivo del área de preparación sin eliminar sus cambios.

git revert <commit>   

Crea un nuevo commit que revierte los cambios de un commit anterior.

git stash          

Guarda temporalmente los cambios no confirmados en un área de almacenamiento (stash).

git stash apply

Aplica los cambios guardados en el stash.

git rebase <rama>     

Reaplica los commits de la rama actual sobre otra base, creando un historial lineal.

git tag <nombre>       

Crea una etiqueta en el commit actual, útil para marcar versiones o lanzamientos.

git config --global user.name "Nombre"           

Configura el nombre de usuario para todos los repositorios Git en tu máquina local.

git config --global user.email "email@example.com"          

Configura el correo electrónico para todos los repositorios Git.

git diff 

Muestra las diferencias entre archivos en el área de trabajo y el área de preparación, o entre commits.

git rm <archivo>        

Elimina un archivo del repositorio y del área de preparación.

git show <commit>    

Muestra los detalles de un commit específico, incluyendo los cambios realizados.
